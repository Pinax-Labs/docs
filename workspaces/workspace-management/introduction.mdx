**Pinaxai Workspaces** are standardized codebases for running Agentic Systems locally using Docker and in production on AWS. They help us manage our Agentic System as code.

![workspace](/images/workspace.png)

## Create a new workspace

Run `pinaxai ws create` to create a new workspace, the command will ask your for a starter template and workspace name.

<CodeGroup>

```bash Create Workspace
pinaxai ws create
```

```bash Create Agent App
pinaxai ws create -t agent-app-aws -n agent-app
```

```bash Create Agent API
pinaxai ws create -t agent-api-aws -n agent-api
```

</CodeGroup>

## Start workspace resources

Run `pinaxai ws up` to start i.e. create workspace resources

<CodeGroup>

```bash terminal
pinaxai ws up
```

```bash shorthand
pinaxai ws up dev:docker
```

```bash full options
pinaxai ws up --env dev --infra docker
```

```bash short options
pinaxai ws up -e dev -i docker
```

</CodeGroup>

## Setup existing workspace

If you clone the codebase directly (eg: if your coworker created it) - run `pinaxai ws setup` to set it up locally

<CodeGroup>



</CodeGroup>

## Command Options

<Note>Run `pinaxai ws up --help` to view all options</Note>

### Environment (`--env`)

Use the `--env` or `-e` flag to filter the environment (dev/prd)

<CodeGroup>

```bash flag
pinaxai ws up --env dev
```

```bash shorthand
pinaxai ws up dev
```

```bash short options
pinaxai ws up -e dev
```

</CodeGroup>

### Infra (`--infra`)

Use the `--infra` or `-i` flag to filter the infra (docker/aws/k8s)

<CodeGroup>

```bash flag
pinaxai ws up --infra docker
```

```bash shorthand
pinaxai ws up :docker
```

```bash short options
pinaxai ws up -i docker
```

</CodeGroup>

### Group (`--group`)

Use the `--group` or `-g` flag to filter by resource group.

<CodeGroup>

```bash flag
pinaxai ws up --group app
```

```bash full options
pinaxai ws up \
  --env dev \
  --infra docker \
  --group app
```

```bash shorthand
pinaxai ws up dev:docker:app
```

```bash short options
pinaxai ws up \
  -e dev \
  -i docker \
  -g app
```

</CodeGroup>

### Name (`--name`)

Use the `--name` or `-n` flag to filter by resource name

<CodeGroup>

```bash flag
pinaxai ws up --name app
```

```bash full options
pinaxai ws up \
  --env dev \
  --infra docker \
  --name app
```

```bash shorthand
pinaxai ws up dev:docker::app
```

```bash short options
pinaxai ws up \
  -e dev \
  -i docker \
  -n app
```

</CodeGroup>

### Type (`--type`)

Use the `--type` or `-t` flag to filter by resource type.

<CodeGroup>

```bash flag
pinaxai ws up --type container
```

```bash full options
pinaxai ws up \
  --env dev \
  --infra docker \
  --type container
```

```bash shorthand
pinaxai ws up dev:docker:app::container
```

```bash short options
pinaxai ws up \
  -e dev \
  -i docker \
  -t container
```

</CodeGroup>

### Dry Run (`--dry-run`)

The `--dry-run` or `-dr` flag can be used to **dry-run** the command. `pinaxai ws up -dr` will only print resources, not create them.

<CodeGroup>

```bash flag
pinaxai ws up --dry-run
```

```bash full options
pinaxai ws up \
  --env dev \
  --infra docker \
  --dry-run
```

```bash shorthand
pinaxai ws up dev:docker -dr
```

```bash short options
pinaxai ws up \
  -e dev \
  -i docker \
  -dr
```

</CodeGroup>

Use the `--debug` or `-d` flag to show debug logs.

<CodeGroup>

```bash flag
pinaxai ws up -d
```

```bash full options
pinaxai ws up \
  --env dev \
  --infra docker \
```

```bash shorthand

```


Use the `--force` or `-f` flag to force recreate images & containers

<CodeGroup>

```bash full options

  --env dev \
```

```bash shorthand

```

```bash short options
pinaxai ws up \
  -e dev \
  -i docker \
  -f
```

</CodeGroup>
